# SIS-111: Fundamentos de Algoritmos y Estructura de Diagramas de Flujo

Esta sesión establece las bases para la resolución de problemas mediante algoritmos y su representación gráfica universal, haciendo hincapié en la precisión técnica y las convenciones de programación.

---

## 1. Definición Refinada de Algoritmo

Un **algoritmo** es una secuencia de pasos finitos, ordenados, coherentes y estructurados destinados a resolver un **problema específico y concreto**.

> [!important] La Especificidad es Clave
> No basta con decir "resuelve problemas". El problema debe estar bien definido. Si un problema es genérico (ej. "la computadora no funciona"), el algoritmo no puede ejecutarse. Se requiere concreción (ej. "la computadora está desenchufada").

### Los 4 Requisitos del Algoritmo
1. **Finitud:** Debe tener un número determinado de pasos (de 1 a 1000) y un final claro.
2. **Coherencia Lógica:** Los pasos deben seguir un orden racional.
   - *Ejemplo:* No puedes sentarte antes de abrir la puerta e ingresar al aula.
3. **Estructura:** Debe prever caminos alternativos (decisiones) según las circunstancias.
4. **Concreción:** Debe apuntar a un resultado único y definido.

---

## 2. Diagramas de Flujo: El Lenguaje Universal

El **diagrama de flujo** es la representación gráfica de un algoritmo o proceso.

- **Universalidad:** Es independiente del lenguaje (Python, Java, etc.). Una lógica bien diagramada puede codificarse en cualquier entorno.
- **Regla de Flujo:** Siempre se construye de **arriba hacia abajo**.
- **Regla de Conectividad:** Todo el diagrama debe estar unido mediante líneas de flujo. No pueden existir elementos "huérfanos".

---

## 3. Simbología y Herramientas Gráficas

| Símbolo | Función | Notas Técnicas |
| :--- | :--- | :--- |
| **Círculo / Óvalo** | Inicio / Fin | Obligatorio. Debe decir "Inicio" al empezar y "Fin" al terminar. |
| **Paralelogramo** | Parámetros de Entrada | Datos que el usuario debe entregar (ej. `L` para el lado). |
| **Rectángulo** | Proceso | Operaciones matemáticas, fórmulas o instanciación. |
| **Rectángulo Curvo** | Salida / Mostrar | Representa el resultado. Tiene una curva en la base para no confundirlo con el proceso. |
| **Rombo** | Condicional | Evalúa una pregunta (Verdad/Falso). |
| **Círculo Pequeño** | Conector | Se usa para continuar el diagrama en otra página (se numera: 1, 2...). |

---

## 4. Lógica de Control (Condicionales)

El **rombo** representa una bifurcación basada en una pregunta que solo admite respuestas binarias (Sí/No, Verdad/Falso).

- **IF-ELSE (Doble vía):** Se realiza una acción por el camino del "Sí" y otra por el del "No".
- **IF Simple (Vía única):** Solo se realiza una acción por el "Sí". Por el "No", la línea de flujo simplemente continúa.
- **Conectividad en Falso:** Aunque no se haga nada por el camino del "Falso", la flecha **debe dibujarse** y conectarse nuevamente al flujo principal para evitar que el algoritmo "se pierda".

## 5. Variables y Nomenclatura Técnica

Una **variable** es un espacio (letra o palabra) que almacena un valor capaz de variar.

### Reglas de Nomenclatura (CamelCase)
1. **Primera letra minúscula**, palabras siguientes con mayúscula (ej. `areaCuadrado`).
2. **Prohibido el uso de acentos:** Aunque en español se usen, en programación (basada en inglés) causan errores de sintaxis.
3. **Claridad:** Nombres que expliquen el contenido (evitar nombres excesivamente largos).

### Instanciación de Variables
> [!important] ¿Por qué instanciar en cero?
> Es obligatorio dar un valor inicial (normalmente 0) a las variables que usaremos. Esto evita errores de **valor nulo (`null`)**. Para una computadora, `null + 10` no tiene resultado, mientras que `0 + 10` sí. Instanciar reduce errores al programar.

---

## 6. Estructura Estándar de un Diagrama (6 Pasos)

Para que un diagrama sea correcto, debe seguir este orden:

1. **Inicio:** Óvalo inicial.
2. **Instanciación:** Rectángulo para declarar variables con valor por defecto (ej. `A = 0`).
3. **Entrada de Datos:** Paralelogramo para capturar los datos necesarios (ej. `L`).
   - *Nota:* Si una variable se pide aquí, no es necesario duplicarla en el paso 2.
4. **Proceso:** Rectángulo con la fórmula o cálculo (ej. `A = L * L`).
5. **Salida:** Símbolo de mostrar resultado.
   - *Tip:* Se puede concatenar texto: `"El área es: " + A`.
6. **Fin:** Óvalo final.
