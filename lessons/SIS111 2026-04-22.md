# SIS-111: Trabajo con `ND` Dígitos y Segmentación de Números

Esta sesión explica cómo dejar de trabajar solo dígito por dígito y empezar a procesar grupos de `ND` dígitos dentro de un número. También se aclara el uso de cadenas de texto (`string`) cuando un ejercicio retorna texto separado por comas, y se dan indicaciones sobre el práctico, el simulacro y el segundo bloque.

## 1. Aclaración Inicial: Número vs. Cadena de Texto

El profesor inicia corrigiendo una confusión: `N` es un número, pero el resultado del ejercicio puede ser una **cadena de texto**.

> [!important] Tipo de dato para texto
> Si el resultado se escribe entre comillas dobles, se está trabajando como texto, es decir, como una cadena (`string`).

La razón es que algunos ejercicios no devuelven simplemente un número, sino una salida con separadores, por ejemplo:

```txt

```

Eso no se maneja igual que un entero puro, porque contiene comas. Por eso se guarda como texto.

> En la transcripción aparece la palabra "stream", pero por el contexto de la explicación se está hablando de una cadena de texto (`string`): letras, textos, números escritos como texto y salidas con comillas dobles.

## 2. Significado de `ND`

`ND` significa **número de dígitos**.

Indica de cuántos dígitos en cuántos dígitos se va a trabajar al extraer partes de un número.

> [!example] Número base
> Supongamos que se trabaja con:
>
> ```txt
> 123456
> ```

### Caso `ND = 1`

Si `ND = 1`, se trabaja de un dígito en un dígito.

Al acceder desde la derecha, se extraen:

```txt
6, 5, 4, 3, 2, 1
```

Esto es lo que ya se venía trabajando en clases anteriores: sacar un dígito con `% 10` y cortar un dígito con `/ 10`.

```cpp
d = n % 10;
n = n / 10;
```

### Caso `ND = 2`

Si `ND = 2`, se trabaja de dos dígitos en dos dígitos.

Con el número `123456`, se extrae:

```txt
56, 34, 12
```

> **Profesor:** Si es de `ND = 2`, ¿de cuántos dígitos tiene que trabajar?
>  
> **Estudiantes:** De dos.
>  
> **Profesor:** Agarró el `56`, agarró el `34` y agarró el `12`.

### Caso `ND = 3`

Si `ND = 3`, se trabaja de tres dígitos en tres dígitos.

Con el número `123456`, se extrae:

```txt
456, 123
```

El problema ya no es solo cortar con `10`; ahora se necesita un valor dinámico que dependa de `ND`.

## 3. Cómo Sacar `ND` Dígitos

Para sacar un dígito se usa:

```cpp
d = n % 10;
n = n / 10;
```

Para sacar dos dígitos, un estudiante propone cambiar `10` por `100`:

```cpp
d = n % 100;
n = n / 100;
```

Esto funciona para `ND = 2`, porque:

```txt
100 = 10^2
```

Para `ND = 3`, se necesitaría:

```txt
1000 = 10^3
```

Por lo tanto, la forma general es:

```cpp
e = pow(10, nd);
d = n % e;
n = n / e;
```

Donde:

- `nd` es la cantidad de dígitos que se quiere extraer.
- `e` representa `10^nd`.
- `d` guarda el grupo extraído.
- `n` se corta quitando ese grupo de dígitos.

> [!important] Fórmula central
> Para trabajar de `ND` dígitos, se debe acceder de `ND` dígitos y también cortar de `ND` dígitos:
>
> ```cpp
> e = pow(10, nd);
> d = n % e;
> n = n / e;
> ```

## 4. Usar Variable o Escribir la Fórmula Directamente

El profesor explica que hay dos maneras válidas de hacerlo.

### Opción 1: Usar una variable

```cpp
e = pow(10, nd);
d = n % e;
n = n / e;
```

Esta opción evita repetir `pow(10, nd)` varias veces.

### Opción 2: Escribirlo directamente

```cpp
d = n % pow(10, nd);
n = n / pow(10, nd);
```

> **Estudiante:** Es más eficiente hacerlo directo.
>  
> **Profesor:** Puede hacerse con variable o sin variable. Es por comodidad. Al final es lo mismo: la variable solo representa `10^nd`.

> [!tip] Recomendación práctica
> Usar una variable como `e` hace que el código sea más legible y evita escribir dos veces la misma expresión. Después se puede cambiar el nombre por algo más claro que `e`.

## 6. Diferencia entre Cortar `ND` Dígitos y Cortar Solo Uno

El profesor recalca que no siempre se corta igual. Depende de lo que pida el ejercicio.

### Caso A: Sacar y cortar de `ND` en `ND`

Si `n = 123456` y `nd = 2`:

```cpp
e = pow(10, nd); // 100

d = n % e; // 56
n = n / e; // 1234

d = n % e; // 34
n = n / e; // 12

d = n % e; // 12
n = n / e; // 0
```

Resultado:

```txt
56, 34, 12
```

### Caso B: Sacar `ND` dígitos, pero cortar solo un dígito

Si se accede de dos dígitos, pero se corta solo con `/ 10`, se genera una ventana desplazada.

Con `n = 123456` y `nd = 2`:

```cpp
e = pow(10, nd); // 100

d = n % e; // 56
n = n / 10; // 12345

d = n % e; // 45
n = n / 10; // 1234

d = n % e; // 34
n = n / 10; // 123

d = n % e; // 23
n = n / 10; // 12

d = n % e; // 12
n = n / 10; // 1
```

Resultado:

```txt
56, 45, 34, 23, 12
```

> [!warning] No es el mismo ejercicio
> Si el enunciado pide `56, 34, 12`, se debe cortar de `ND` en `ND`.
> Si el enunciado pide `56, 45, 34, 23, 12`, se debe cortar de uno en uno.

> **Profesor:** ¿Y acaso está saliendo así?
>  
> **Estudiantes:** No.
>  
> **Profesor:** Pero si en otro ejercicio de examen sale `56, 45, 34, 23 y 12`, ahí sí estaría bien. Tienen que darse cuenta de qué les está pidiendo el ejercicio.

## 7. Cuando Sobran Menos Dígitos que `ND`

También se habló del caso en el que la cantidad de dígitos del número no encaja exactamente con `ND`.

Si `ND = 2` y al final queda un solo dígito, por ejemplo `8`, se toma como:

```txt
08
```

Pero como los ceros a la izquierda no se muestran, en la salida aparece simplemente:

```txt
8
```

Si `ND = 3`, ese mismo dígito se puede entender como:

```txt
008
```

De nuevo, los ceros iniciales no se ven.

> [!example] Dígitos faltantes
> Si queda un grupo incompleto al final, el programa no muestra los ceros de relleno. Solo se ve el valor real que queda.

> **Profesor:** Si es de dos, agarra simplemente el `8`.
>  
> **Profesor:** Ese último dígito es como si tuviera un `08`.
>  
> **Profesor:** Si fuera de tres, sería como `008`, pero los ceros no se ven.

## 11. Resumen Técnico de la Clase

| Objetivo | Operación | Resultado |
| :--- | :--- | :--- |
| Sacar 1 dígito | `d = n % 10` | Último dígito |
| Cortar 1 dígito | `n = n / 10` | Quita el último dígito |
| Sacar `ND` dígitos | `d = n % pow(10, nd)` | Último grupo de `ND` dígitos |
| Cortar `ND` dígitos | `n = n / pow(10, nd)` | Quita el último grupo de `ND` dígitos |
| Sacar grupos solapados | `d = n % e`, luego `n = n / 10` | Ventanas como `56, 45, 34...` |
| Sacar grupos no solapados | `d = n % e`, luego `n = n / e` | Grupos como `56, 34, 12` |
