# SIS-111: Intercalación de Dígitos Pares e Impares

Esta sesión desarrolla una técnica de intercalación de dígitos: formar un nuevo número alternando dígitos pares e impares, respetando el orden pedido para cada grupo. La idea importante no es memorizar el código, sino entender cómo consumir los dígitos paso a paso y luego llevar esa lógica a programación.

---

## 1. Problema Principal

**Consigna:** Intercalar los dígitos pares e impares de un número, ordenados descendentemente.

La salida debe empezar con:

1. El mayor dígito par.
2. El mayor dígito impar.
3. El siguiente mayor dígito par.
4. El siguiente mayor dígito impar.
5. Y así sucesivamente hasta que ya no queden dígitos.

Si uno de los dos grupos se acaba primero, se siguen agregando los dígitos restantes del otro grupo.

> [!example] Idea de resultado
> Si los pares disponibles son `6, 4, 2` y los impares son `7, 5, 5, 3, 1`, el nuevo número se forma así:
> `6 7 4 5 2 5 3 1`

---

## 2. Observación Clave: Ordenar Antes de Intercalar

Para sacar dígitos de un número se usa:

```cpp
digito = n % 10;
n = n / 10;
```

Esto siempre extrae primero el dígito de la derecha.

Por eso, aunque el resultado pide los dígitos en orden **descendente**, la estrategia usada fue ordenar el número de forma **ascendente** antes de intercalar.

> [!important] ¿Por qué ascendente?
> Si el número está ordenado como `1234567`, al sacar con `% 10` se obtiene primero `7`, luego `6`, luego `5`, etc. Es decir, se pueden encontrar primero los dígitos mayores.

---

## 3. Estrategia General

La técnica se basa en trabajar con dos copias del número ordenado:

- `numeroCopiaPar`: sirve para buscar el siguiente dígito par.
- `numeroCopiaImpar`: sirve para buscar el siguiente dígito impar.

Se usan dos copias porque al buscar un par se van cortando dígitos. Si aparece un impar durante esa búsqueda, ese impar se descarta de la copia de pares, pero todavía debe existir en la copia de impares para poder usarse después.

### Pasos del algoritmo

1. Ordenar el número ascendentemente.
2. Copiar el número ordenado en `numeroCopiaPar` y `numeroCopiaImpar`.
3. Mientras al menos una copia tenga dígitos:
   - Buscar el primer dígito par disponible.
   - Si se encuentra, agregarlo al nuevo número.
   - Buscar el primer dígito impar disponible.
   - Si se encuentra, agregarlo al nuevo número.
4. Asignar el nuevo número a `n`.

> [!warning] Usar `OR`, no `AND`
> El ciclo principal debe continuar mientras `numeroCopiaPar > 0 OR numeroCopiaImpar > 0`.
> Si se usa `AND`, el algoritmo se detiene cuando una copia se queda sin dígitos, aunque todavía queden dígitos en la otra.

---

## 4. Variable `encontre`

Se usa una variable booleana para detener la búsqueda apenas se encuentra el primer dígito válido.

- Antes de buscar: `encontre = false`.
- Cuando se encuentra el dígito buscado: `encontre = true`.
- El ciclo interno trabaja mientras:

```cpp
numeroCopia > 0 && encontre == false
```

> [!important] Reiniciar `encontre`
> Antes de buscar un par y antes de buscar un impar, `encontre` debe volver a `false`.
> Si no se reinicia, el segundo ciclo puede no ejecutarse.

---

## 5. Procedimiento para Ordenar Ascendentemente

Una forma de ordenar los dígitos de un número de menor a mayor es recorrer los posibles dígitos del `1` al `9` y reconstruir el número.

```cpp
void ordenarAscendente(int &n) {
    int copia = n;
    int nuevoNumero = 0;
    int digito;

    for (int i = 1; i <= 9; i++) {
        n = copia;

        while (n != 0) {
            digito = n % 10;
            n = n / 10;

            if (digito == i) {
                nuevoNumero = nuevoNumero * 10 + digito;
            }
        }
    }

    n = nuevoNumero;
}
```

> [!note] Paso final obligatorio
> Al terminar el ordenamiento, se debe hacer `n = nuevoNumero`.
> Si se olvida ese paso, el número original queda destruido porque durante el proceso se fue dividiendo entre `10`.

## 6. Procedimiento de Intercalación

```cpp
void intercalarParImparDescendente(int &n) {
    int nuevoNumero = 0;
    int numeroCopiaPar;
    int numeroCopiaImpar;
    int digito;
    bool encontre;

    ordenarAscendente(n);

    numeroCopiaPar = n;
    numeroCopiaImpar = n;

    while (numeroCopiaPar > 0 || numeroCopiaImpar > 0) {
        encontre = false;

        while (numeroCopiaPar > 0 && encontre == false) {
            digito = numeroCopiaPar % 10;
            numeroCopiaPar = numeroCopiaPar / 10;

            if (digito % 2 == 0) {
                nuevoNumero = nuevoNumero * 10 + digito;
                encontre = true;
            }
        }

        encontre = false;

        while (numeroCopiaImpar > 0 && encontre == false) {
            digito = numeroCopiaImpar % 10;
            numeroCopiaImpar = numeroCopiaImpar / 10;

            if (digito % 2 != 0) {
                nuevoNumero = nuevoNumero * 10 + digito;
                encontre = true;
            }
        }
    }

    n = nuevoNumero;
}
```

### Llamada desde el programa principal

```cpp
int n;

cout << "Ingrese un numero: ";
cin >> n;

intercalarParImparDescendente(n);

cout << "El nuevo numero es: " << n;
```

> [!important] Paso final obligatorio
> Como el procedimiento modifica `n`, debe recibirlo por referencia con `int &n`.
> Al final se asigna `n = nuevoNumero` para devolver el resultado.

## 7. Errores Frecuentes

1. **Ordenar en la dirección incorrecta.**
   - Para obtener los mayores primero usando `% 10`, conviene tener el número ascendente.

2. **Usar una sola copia del número.**
   - Si se usa una sola copia, al buscar pares se pierden impares, o viceversa.

3. **Usar `&&` en el ciclo principal.**
   - Debe ser `||` porque basta con que una copia todavía tenga dígitos.

4. **No reiniciar `encontre`.**
   - Cada búsqueda nueva debe comenzar con `encontre = false`.

5. **Olvidar formar el nuevo número.**
   - Para agregar un dígito a la derecha:

```cpp
nuevoNumero = nuevoNumero * 10 + digito;
```

6. **Olvidar devolver el resultado.**
   - Al final del procedimiento:

```cpp
n = nuevoNumero;
```

## 8. Generalización de la Técnica

El profesor remarcó que esta técnica sirve para muchas variantes de intercalación. Lo que cambia normalmente es la condición.

### Ejemplos de variantes

- Intercalar hombres y mujeres.
- Intercalar aprobados y aplazados.
- Intercalar estudiantes con lentes y sin lentes.
- Intercalar dígitos primos y no primos.
- Intercalar pares ascendentes e impares descendentes.
- Intercalar dígitos Fibonacci y no Fibonacci.

La estructura general se mantiene:

1. Preparar el orden necesario.
2. Tener una copia para cada grupo.
3. Buscar el siguiente elemento del primer grupo.
4. Buscar el siguiente elemento del segundo grupo.
5. Formar el nuevo resultado.

### Caso: dígitos primos

Para saber si un dígito es primo, se puede preguntar directamente:

```cpp
digito == 2 || digito == 3 || digito == 5 || digito == 7
```

> [!warning] Usar `OR`, no `AND`
> Un dígito no puede ser `2`, `3`, `5` y `7` al mismo tiempo.
> Por eso se usa `||`, porque basta con que sea uno de esos valores.

## 9. Idea Central de la Clase

El profesor insistió en que el objetivo no es memorizar el código.

Primero se debe entender el proceso sin programar:

1. ¿Qué dígito busco?
2. ¿En qué copia lo busco?
3. ¿Qué hago cuando lo encuentro?
4. ¿Cuándo dejo de buscar?
5. ¿Cuándo paso al otro grupo?

Una vez clara esa lógica, convertirla en código es más sencillo.

> [!quote] Enfoque recomendado
> No ver directamente el código. Primero entender cómo llegar al resultado paso a paso; después recién traducirlo a programación.
